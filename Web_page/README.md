# 📦 Folder — `Web_page`

The `pagina_web` folder contains the fundamental files required to configure and run the development and deployment environment of the system. It includes:

* `manage.py`
* `package.json`
* `package-lock.json`
* `requirements.txt`

---

## 1) `manage.py` — Django

### 🧠 Description

`manage.py` is a Django utility file used to execute administrative tasks such as database migrations, superuser creation, application creation, and general project management.

### ⚙️ Generation

This file is created automatically when a new Django project is initialized with the following command:

```bash
# Creates a new Django project named robotat_web.
django-admin startproject robotat_web
```

This process generates the following structure:

```text
backend/
├─ manage.py
└─ robotat_web/
   ├─ settings.py
   ├─ urls.py
   ├─ asgi.py
   └─ wsgi.py
```

The `manage.py` file sets the `DJANGO_SETTINGS_MODULE` environment variable to point to `robotat_web.settings`.

### ▶️ Main Commands

```bash
# Applies database migrations to create or update database tables.
python manage.py migrate

# Creates an administrator account for the Django admin interface.
python manage.py createsuperuser

# Creates a new Django application inside the project.
python manage.py startapp app_name

# Runs the Django development server.
python manage.py runserver
```

### 💡 Running with Daphne

In this project, the server is executed using Daphne, which uses the ASGI interface:

```bash
# Runs the Django ASGI application on port 8000 using Daphne.
daphne -p 8000 robotat_web.asgi:application
```

The `manage.py` file is still required for all administrative operations within the Django environment.

## 2) `package.json` — Node.js / npm

### 🧠 Description

`package.json` is a configuration file that defines the JavaScript environment of the project. It contains the dependencies, scripts, and metadata required for the frontend implemented with React and Vite.

### ⚙️ Generation

It is generated when a project is initialized with npm:

```bash
# Initializes a new npm project with default values.
npm init -y
```

For a React environment using Vite and TypeScript, the following commands are used:

```bash
# Creates a new React + TypeScript frontend project using Vite.
npm create vite@latest frontend -- --template react-ts

# Moves into the frontend project directory.
cd frontend

# Installs the frontend dependencies defined in package.json.
npm install
```

This creates a `package.json` file inside the `frontend` folder with the corresponding configuration.

### ➕ Installing Dependencies

```bash
# Installs the main React dependencies required to build the frontend interface.
npm install react react-dom

# Installs development dependencies for TypeScript, Node.js types, and Vite.
npm install -D @types/node typescript vite
```

### ▶️ Main Scripts

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  }
}
```

Execution commands:

```bash
# Starts the Vite development server.
npm run dev

# Builds the frontend application for production.
npm run build

# Serves a local preview of the production build.
npm run preview
```

## 3) `package-lock.json` — npm Lockfile

### 🧠 Description

`package-lock.json` is an autogenerated file created by npm. It records the exact versions of the installed dependencies and ensures that installations are reproducible across different environments.

### ⚙️ Generation

It is automatically created or updated when dependencies are installed:

```bash
# Installs project dependencies and updates package-lock.json if needed.
npm install
```

### 🔁 Reproducible Installation

```bash
# Installs dependencies using the exact versions recorded in package-lock.json.
npm ci
```

The `npm ci` command installs dependencies strictly according to the versions registered in `package-lock.json`.

## 4) `requirements.txt` — Python

### 🧠 Description

`requirements.txt` lists the dependencies required to run the Django backend. It contains the Python packages and versions used by the project.

### ⚙️ Generation

Inside the virtual environment, after installing the required packages:

```bash
# Installs the Python packages required for the Django backend and supporting services.
pip install django djangorestframework djangorestframework-simplejwt daphne requests numpy opencv-python

# Saves the installed Python package versions into requirements.txt.
pip freeze > requirements.txt
```

To install the registered dependencies:

```bash
# Installs all Python dependencies listed in requirements.txt.
pip install -r requirements.txt
```

### 📦 Main Packages

* `Django`, `djangorestframework`, and `djangorestframework-simplejwt`: API development and authentication.
* `daphne`: ASGI server.
* `requests`, `numpy`, and `opencv-python`: data processing and communication.

---

## ✅ Summary

* `manage.py`: generated by `startproject`; used to execute Django administrative tasks.
* `package.json`: generated by `npm init`; defines dependencies and scripts for the JavaScript environment.
* `package-lock.json`: created by `npm install`; locks exact dependency versions.
* `requirements.txt`: generated with `pip freeze`; contains the Python dependencies required by the project.
